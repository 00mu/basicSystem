## HTTP

### 1. 主要特点
特点：灵活可拓展、无连接（结束即断开）、无状态（不记录上下文信息）

缺点：无状态（既是优点又是缺点）、明文传输、队头阻塞问题

### 2. 报文的组成部分

报文有请求报文和响应报文组成。其中请求报文包含：请求行（HTTP方法、页面地址）、请求头（key-value形式带上一些信息）、空行（分割请求头和请求体）、请求体；响应报文对应包含：状态行，响应头、空行、响应体；

### 3. HTTP请求方法有哪些

get、post、push、delete、head

### 4.get和post的区别

1. get请求在浏览器回退时不会重复请求；post会再次提交
2. get请求会被浏览器自动缓存；post不会
3. get请求通过url传参；post放在request body中
4. get请求的参数会完整保存在浏览器的历史记录中同时可以被收藏；post不会
5. get请求比post更不安全，因为参数暴露在url上，所以不能传递敏感信息
6. get请求只接受ascii编码；post没有限制
7. get请求的参数长度（2k）有限制；post没有限制

### 5. 状态码

1. 1XX：指示信息，表请求已接受，继续处理
2. 2XX：传递成功，表信息已被成功接受。200（客户端请求成功）、206（客户端发送一个带有range头的get请求，用于比较大的文件video、audio）
3. 3XX：重定向，要完成请求必须进行进一步操作。301（请求转移到新的url）、302（请求临时转移到新的url）、304（客户端有缓存，并发出一个条件性的请求，服务端告诉客户端原缓存文档可以继续使用）
4. 4XX：客户端错误，表请求语法错误或者无法实现。400（语法错误不被服务端理解）、401（未经授权）、403（请求的页面访问被禁止，通常只能服务端访问）、404（请求的资源不存在）
5. 5XX：服务端的错误，表服务端未能实现合法的请求。500（服务端发生不可预期的错误，原缓存还可以继续使用）、503（服务器临时过载或宕机）

### 跨域

同源策略：非同源（协议、主机、端口三者都相同为同源），不能读取对方的dom、cookie、indexdb、localstorage；限制XMLHTTPRequest请求。

Ajax跨域是响应被浏览器拦截，响应返回后主进程检查到跨域且没有cors响应头，就将响应体全部丢掉，不发送给渲染进程，达到拦截目的。

跨域的方案：

1. jsonp（script标签+回调函数）
2. cors（原生跨域）
3. nginx(反向代理)
4. WebSocket
